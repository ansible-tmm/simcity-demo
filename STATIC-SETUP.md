
# Static Site Setup

This site has been converted to run as a static HTML/JavaScript application without requiring PHP. It works with both HTTP servers and the `file://` protocol (local files).

## How It Works

The site uses embedded JavaScript data instead of PHP endpoints to look up files and folders. All data is embedded in `assets/data.js`, which is generated by running a PHP prepare script. This eliminates CORS issues when opening the site directly from the file system.

## Setup Instructions

### 1. Generate Static JSON Files

Before deploying the static site, run the prepare script to generate all required JSON files:

```bash
php prepare-static.php
```

This script will:
- Scan the entire `assets/images/` directory structure
- Generate JSON files (`folders.json`, `images.json`, `media.json`) for each directory (stored in `data/`)
- Generate a single JavaScript file (`assets/data.js`) containing all data embedded as a JavaScript object
- Find all 360° videos and encode each as base64 in its own file under `assets/360/*.js`, plus a small `assets/videos360-manifest.js` (for file:// protocol; scripts are loaded on demand)

### 2. Deploy Static Files

After running the prepare script, you can deploy:
- All HTML, CSS, and JavaScript files
- The `assets/` directory (images, videos, `data.js`, `videos360-manifest.js`, and `assets/360/` per-video scripts)
- The `data/` directory (generated JSON files - optional, only needed if you want to keep them)

**Note:** 
- The PHP files (`get-folders.php`, `get-images.php`, `get-media.php`) are no longer needed for the static site to function.
- The site will work with `file://` protocol (opening `index.html` directly) because all data is embedded in `assets/data.js` and 360° video scripts are loaded on demand from `assets/360/` via `assets/videos360-manifest.js`.
- Only the 360° video(s) you view (and the format chosen in the admin modal) are loaded; each per-video file can be large (tens to 100MB+).

## Regenerating JSON Files

If you add, remove, or modify files in `assets/images/`, you'll need to regenerate the JSON files:

```bash
php prepare-static.php
```

The script will overwrite existing JSON files with updated data.

## How JavaScript Loads Data

The JavaScript code now reads data from the embedded `staticData` object in `assets/data.js` instead of making PHP AJAX calls:

- **Folders**: `staticData[folderPath].folders`
- **Images**: `staticData[folderPath].images`
- **Media**: `staticData[folderPath].media`

This approach:
- ✅ Works with HTTP servers
- ✅ No network requests needed for data (all data is embedded)
- ✅ Faster loading (no AJAX delays)

**360° Videos Support:**
- ✅ 360° videos are encoded as base64 during the prepare process (one `.js` file per video in `assets/360/`)
- ✅ They work with `file://` protocol using blob URLs (no web server needed!)
- ✅ Only the video(s) you view—and the format selected in the admin modal—are loaded at runtime
- ⚠️ Each per-video file in `assets/360/` can be large (tens to 100MB+) depending on video size

## Running with a Local Web Server (Optional)

A local web server is **not required** - the site works fully with `file://` protocol! However, you can still use a web server if preferred:

### Option 1: Use the provided script
```bash
./start-server.sh
```
Then open: `http://localhost:8000`

### Option 2: Python
```bash
python3 -m http.server 8000
# or
python -m SimpleHTTPServer 8000
```
Then open: `http://localhost:8000`

### Option 3: PHP
```bash
php -S localhost:8000
```
Then open: `http://localhost:8000`

The `data/` directory with JSON files is still generated for reference, but the JavaScript uses the embedded data from `assets/data.js`.
